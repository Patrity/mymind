# Cycle 18 — Voice Agent v2 (handover)

Replaced the Unmute/Kyutai-orchestrated voice path (cycle 17) with a fully self-owned TypeScript pipeline. The Vercel AI SDK is now the agent core shared by voice, text chat, and future cron agents. Audio is local OpenAI-spec (faster-whisper STT, Kokoro/Chatterbox TTS) over plain WebSockets.

## What shipped

### Architecture

One WebSocket (`/api/voice/ws`) replaces the two-connection cycle-17 design (Unmute WS + SSE side-channel).

```
Browser: Silero VAD → WAV utterance → WS → Orchestrator → STT → runAgent → TTS → WAV → WS → browser
```

The four layers (client voice UI, Nitro orchestrator, provider adapters, agent core) are cleanly separated — audio/VAD concerns never touch the agent core.

### AI SDK agent core (`server/lib/agent/run.ts`)

`runAgent` wraps `streamText` from the Vercel AI SDK with `@ai-sdk/openai-compatible` pointed at the local qwen reasoning model. It yields a typed `AgentEvent` stream (`text-delta`, `tool-start`, `tool-result`, `done`). The tool registry adapts to AI SDK `tool()` via a small `toAiSdkTools()` adapter — the registry (`tools.ts`) remains the single source of truth and is unchanged from cycle 17.

Replaces the hand-rolled `loop.ts` + `streamChat` + Zod→JSON-schema mapper. Less code, no custom protocol, proper SDK type safety.

### Unmute teardown

`/api/agent/llm` (the Unmute shim endpoint), `loop.ts`, `chat-stream.ts`, `openai-chunk.ts`, `useUnmute.ts`, `opus-recorder` dep, and `public/opus/` are all removed. The `NUXT_PUBLIC_UNMUTE_URL` env var and the LAN-only middleware exemption for `/api/agent/llm` are gone.

### Key decisions

1. **Own the loop in TS** — the Unmute approach bought barge-in for free but locked us to Kyutai's model, binary msgpack protocol, global LLM config, and rig-side tuning. Owning the orchestrator in Nitro means dev/prod coexist, all tuning is a constant in `tuning.ts`, and the model is a base_url env change.

2. **Vercel AI SDK over hand-rolled** — `streamText` gives us tool-calling, AbortSignal, step counting, and stream consumption out of the box. Cycle 17's loop was ~200 lines re-implementing what the SDK does correctly and with type safety.

3. **Client-VAD barge-in** — Silero VAD (`@ricky0123/vad-web`) runs in the browser on a dedicated AudioWorklet. The client detects speech onset during playback, stops audio instantly, and sends `{type:'interrupt'}` over the WS. The server aborts `runAgent` via AbortSignal. Echo cancellation (`echoCancellation: true`) on the mic prevents the assistant's own audio from false-triggering barge-in.

4. **Swappable local providers over OpenAI-spec** — both STT (faster-whisper) and TTS (Kokoro, Chatterbox) expose standard OpenAI audio endpoints. A new provider = a new adapter file or just a new base_url. No binary protocol, no vendor lock-in.

## Manual cutover steps

1. **Set env vars** — add the cycle-18 block to your dev `.env` and to the prod LXC 114 `.env` (see `.env.example` `# Voice agent (cycle 18)` block). Specifically: `AI_STT_MODEL`, `AI_TTS_KOKORO_BASE_URL`, `AI_TTS_KOKORO_MODEL`, `AI_TTS_KOKORO_VOICE`, `AI_TTS_CHATTERBOX_BASE_URL`, `AI_TTS_CHATTERBOX_MODEL`, `AI_TTS_CHATTERBOX_VOICE`.

2. **Unmute is no longer used** — the rig's Unmute container (`192.168.2.25`) is no longer part of the voice path. You can stop it or leave it idle; no config changes are needed on it (no LLM repoint required as it was in cycle 17).

3. **Restart / redeploy** — `pnpm dev` for local. Redeploy prod (LXC 114): pull latest image, restart container with updated env.

4. **Manual voice acceptance** — requires HTTPS or localhost (mic secure-context). Open `/voice`, confirm the VAD indicator activates, speak a test utterance, verify transcript appears and TTS audio plays back. Test barge-in: start speaking while the assistant is responding. Tune `VOICE_TUNING` constants in `server/lib/voice/tuning.ts` as needed (no rebuild — dev server hot-reloads).

5. **Chatterbox voice param** — if you switch to Chatterbox as the active provider, ensure `AI_TTS_CHATTERBOX_VOICE` is set (a 422 is returned if the voice param is omitted from the request).

## Validation

`pnpm typecheck` + `pnpm build` pass. AI SDK ↔ qwen tool-calling confirmed end-to-end (typed `/voice` path in dev). STT ↔ TTS round-trip confirmed against the rig (Kokoro/Chatterbox WAV out, Whisper transcribes back exactly). Full audio voice loop (turn-taking, barge-in feel) requires manual acceptance over HTTPS/localhost with a real microphone.
